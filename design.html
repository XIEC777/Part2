01

产品愿景

我们解决什么问题

现代学习者在碎片化时间中面临两大挑战：一是无法有效进入专注状态，二是长时间专注后产生倦怠感。传统的番茄钟工具仅提供单一的计时功能，缺乏对学习者心理状态的关注。

核心假设

不同 MBTI 人格类型对环境氛围的偏好存在显著差异。一个匹配用户人格特质的感官环境——包括视觉、听觉和节奏——可以更有效地触发心流状态，提升专注效率与学习体验。

设计原则

以人为本 — 工具适应人，而非人适应工具。每一种人格都有属于自己的学习空间。

产品定位

🍅
专注计时器
番茄钟 + 短休 + 长休，支持自定义时长
🌌
沉浸空间

🎵
环境音景
可混音的自然环境音 + 生成式旋律背景
🏆

里程碑解锁 + 连续打卡 + 学习轨迹记录

音频设计

架构概览

Web Audio API AudioContext │ ├── Master Gain (主音量, 0 ~ 0.7) │ │ │ ├── Channel: Rain [BufferSource → Bandpass(3000, Q:0.5) → Gain] │ ├── Channel: Fire [BufferSource → Lowpass(400, Q:1.0) → Gain] │ ├── Channel: Wind [BufferSource → Bandpass(400, Q:0.3) → Gain] │ ├── Channel: Cafe [BufferSource → Bandpass(1200, Q:0.8)→ Gain] │ ├── Channel: Forest [BufferSource → Bandpass(2000, Q:0.5)→ Gain] │ │ │ ├── Music: Note [Oscillator → Gain → Convolver + Master] │ ├── Music: Pad [Oscillator → Gain → Convolver + Master] │ │ │ └── Bell [Oscillator × 3 → Gain → Master] │ └── destination

7.1 环境噪音合成

所有环境音基于一段 4 秒的棕色噪声缓冲区 (Brown Noise) 合成，而非外部音频文件。棕色噪声由 Voss-McCartney 算法生成，通过不同的滤波器配置模拟不同的自然声景。

设计决策

采用合成噪声而非录音，原因：1) 零文件体积，首屏加载极快；2) 无缝循环，无拼接点；3) 精确控制频段分布，适应不同场景需求。

7.2 生成式背景音乐

背景音乐使用纯合成方式，由以下部分组成：

组件实现描述
音阶4 种调式NT: 小调含增四度 | NF: 自然大调 | SJ: 低音域大调 | SP: 含变化音
旋律层Sine Oscillator随机音阶音符，0.7~1.3x 节奏间隔。63%~0% 指数衰减信封。
Pad 层Sine Oscillator低八度双音。较长的衰减时间。间隔更大。
混响Convolver2.5 秒指数衰减脉冲响应，模拟空间感。28% 湿信号混合。

7.3 音符信封

每个音符使用三段式信封，确保干净的起音与止音：

Attack: 0.06s 线性渐入 (LinearRamp 0 → peak) Sustain: peak 维持至音符时长的 60% Release: 指数衰减 (SetTargetAtTime → 0.0001, τ = dur×0.25) Stop: 强制静音 + Oscillator.stop() (释放后 2 秒)

09

交互设计

9.1 页面过渡

选择页 → 自习室：选择页通过 opacity fade + scale(1.03) 向后退隐 (0.8s)，同时自习室从透明淡入 (0.8s delay 0.3s)，形成「前层消失，后层浮现」的视觉叙事。

自习室 → 选择页：通过切换按钮触发。自习室 fadeOut，选择页 fadeIn 并重置滚动位置。所有音频静音，计时器停止。

9.2 计时器交互

点击 ▶ 开始
→
显示 ⏸ (可暂停)
→
完成时播放铃声
→
弹出完成通知
→
用户关闭 → 恢复计时器

9.3 音频初始化策略

浏览器要求用户交互后才能创建 AudioContext。因此采用「延迟初始化」策略：AudioContext 在首次用户交互时才创建。

实现细节

音乐按钮点击时检测 audioContext.state === 'suspended'，若为 suspended 则先调用 resume() 再执行后续逻辑。所有音频操作包裹在 try-catch 中，确保音频初始化失败不影响其他功能。

9.4 键盘快捷键

按键功能条件
Space开始 / 暂停计时不在输入框内
R重置计时器不在输入框内
F切换专注模式不在输入框内
Esc依次关闭：完成通知 → 面板 → 专注模式—

9.5 指针事件管理

自习室 UI 层 (#studyUI) 初始状态设置 pointer-events: none，确保选择页的交互不受干扰。仅当添加 .show 类 (opacity > 0) 后，才启用 pointer-events: auto。所有交互性按钮额外设置 z-index: 12+ 和 position: relative，确保在 backdrop-filter 创建的层叠上下文中仍可被点击。

10.3 DOM 缓存策略

所有 DOM 引用在 DOMContentLoaded 回调内一次性缓存到 dom 对象中，避免运行时重复查询：

var dom = {}; dom.clock = $('clock'); dom.tTime = $('tTime'); dom.btnGo = $('btnGo'); // ... (40+ 引用一次性缓存)

10.4 事件注册策略

所有事件监听器在 boot() 函数中统一注册，使用 addEventListener。关键按钮添加 e.preventDefault() + e.stopPropagation() 防止事件冒泡干扰。

var ST = { mbti: String, // 当前选择的人格类型 ("INTJ" | null) timer: { mode: String, // "focus" | "short" | "long" total: Number, // 当前模式总秒数 remain: Number, // 剩余秒数 running: Boolean, // 是否正在计时 iv: Number, // setInterval ID sessions: Number // 累计完成番茄数 }, todos: Array, // [{ text, done }] journal: Array, // [{ text, time }] achievements: Array, // ["first", "ten", ...] vol: Number, // 主音量 0~100 focusMin: Number, // 专注时长 (分钟) dailyGoal: Number, // 每日番茄目标 focus: Boolean, // 专注模式开关 eyeRest: Boolean, // 护眼提醒开关 history: Object, // { "2025-06-15": 4, ... } theme: String, // "dark" | "light" transitioning: Boolean // 正在切换场景 };

11.2 持久化数据 (localStorage)

存储键名: mbti_room_v10，格式为 JSON 字符串。

字段类型说明
mbtiString上次选择的人格类型
sessionsNumber累计番茄数
todosArray待办列表
journalArray学习日记
achArray已解锁成就 ID
volNumber音量设置
fmNumber专注时长偏好
dgNumber每日目标
historyObject日期→完成数映射
eyeRestBoolean护眼开关
themeString主题偏好
